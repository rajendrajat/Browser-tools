Tests use fetch() with mode: 'no-cors' + HEAD requests from your browser. A result of Reachable means the network path is open (your firewall/proxy let the packet through). Blocked means a connection-level failure — the endpoint is unreachable from this network. Redirected means the server responded but forwarded elsewhere (common for login flows). Note: no-cors opaque responses always return status 0 — this is normal for non-CORS APIs; reachability is still confirmed.
